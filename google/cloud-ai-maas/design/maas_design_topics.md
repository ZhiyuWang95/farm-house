# MaaS System Design Practice — Candidate Prompts

To be worked through once a loop is scheduled. Pulled from the team's stated
responsibilities in `../job_description/role_overview.md`.

- [ ] Design a multi-tenant model-serving platform that lets external partners
  (e.g., Anthropic, Mistral) onboard their own models and manage their service
  configuration.
- [ ] Design a prefix-cache-aware routing layer for a large multi-replica
  inference deployment.
- [ ] Design a metering/billing pipeline for usage-based monetization of
  partner-hosted models.
- [ ] Design an onboarding pipeline that takes a new open-source model (e.g., a
  Llama checkpoint) from upload to a servable, autoscaled endpoint.
- [ ] Design the capacity/placement system for scheduling large model replicas
  across GPU/TPU pools with mixed model sizes and SLAs.
